    slot

    0x0::slot

    Module for an NFT release Slot

    After the creation of the Launchpad a Slot for the NFT release needs to be created. Whilst the Launchpad stipulates a default fee policy, the launchpad admin can decide to create a custom fee policy for each release Slot.

    The slot acts as the object that configures the primary NFT release strategy, that is the primary market sale. Primary market sales can take many shapes, depending on the business level requirements.

    Structs

    NftCertificate

    0x0::slot::NftCertificate

    This object acts as an intermediate step between the payment and the transfer of the NFT. The user first has to call buy_nft_certificate which mints and transfers the NftCertificate to the user. This object will dictate which NFT will the user receive by calling the endpoint claim_nft

    id object::UID

    Launchpad ID to which the Slot this certificate is assigned

    Intended for discoverability
    slot_id object::ID

    Slot from which this certificate can withdraw an Nft
    nft_id object::ID

    ID of the Nft which can be withdrawn using this certificate

    WhitelistCertificate

    0x0::slot::WhitelistCertificate

    Whitin a release Slot, each market has its own whitelist policy. As an example, creators can create tiered sales based on the NFT rarity, and then whitelist only the rare NFT sale. They can then emit whitelist tokens and send them to users who have completed a set of defined actions.

    Fields:
    Name Type Description
    id object::UID
    launchpad_id object::ID

    Launchpad ID intended for discoverability
    slot_id object::ID

    Slot from which this certificate can withdraw an Nft
    market_id object::ID

    Inventory from which this certificate can withdraw an Nft
